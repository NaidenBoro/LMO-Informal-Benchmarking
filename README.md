# LMO Informal Benchmarking

Code for the thesis **Leave-Multiple-Out Informal Benchmarking: Understanding the Behaviour of Informal Benchmarking of Multivariate Confounding**.

The project studies whether leave-multiple-out (LMO) informal benchmarking recovers the true strength of omitted-variable confounding as more observed covariates are treated as hidden. The main takeaway is interpretive: the empirical LMO benchmark is a sample-realized bound. It depends not only on the structural treatment weights, but also on whether the observed sample contains individuals near the covariate configurations that generate large odds-ratio shifts.

## Method In Brief

All experiments use a controlled synthetic treatment-assignment setting:

- `p = 10` bounded covariates with values in `[-1, 1]`.
- Structural treatment weights `w = (1.0, 0.9, ..., 0.1)`.
- Treatment probabilities generated by `expit(X @ w)`.
- Binary treatment sampled from the resulting Bernoulli probabilities.
- LMO benchmarking over every feature subset of size `m = 1, ..., 10`.

For each dropped subset, the scripts compare three quantities:

- **Empirical LMO Gamma**: the practical informal benchmark. It fits a full logistic propensity model, fits a reduced model after dropping the subset, and takes the largest individual log-odds shift observed in the sample.
- **Oracle Gamma**: a sample-realized structural benchmark. It uses the known dropped contribution `X_S @ w_S`, so it separates finite-sample geometry from propensity-model estimation error.
- **Theoretical Gamma**: the maximum possible structural bound over the full bounded covariate space, computed as `exp(sum of the m largest absolute weights)`. This is attainable only at aligned corners of the covariate space.

## Experiments

### Experiment 1: Aggregate LMO Recovery

`exp1.py` compares empirical, oracle, and theoretical Gamma curves for uniform covariates at `N = 1,000`, `10,000`, and `100,000`.

It tests the central pattern from the thesis: the theoretical ceiling increases monotonically as more covariates are omitted, but the empirical and oracle sample-realized bounds can plateau or decline because the required aligned corner observations are rare.

Outputs:

- `outputs/experiment1_raw.csv`
- `outputs/experiment1_summary.csv`
- `outputs/reversal_hook_summary.csv`
- `outputs/reversal_hook_summary.tex`
- `figures/oracle_vs_empirical.png`
- `figures/recovery_ratio.png`

### Experiment 2: Individual Trajectories

`exp2.py` tracks each individual's strongest fitted Gamma value across all dropped subsets of each size.

This shows that the aggregate LMO maximum is not typical of the sample. It is controlled by a small number of boundary individuals whose omitted covariates align in sign and magnitude with the treatment weights.

Outputs:

- `outputs/experiment2_summary.csv`
- `outputs/experiment2_top_individuals.csv`
- `outputs/experiment2_individual_trajectories.csv`
- `figures/individual_gamma_distribution.png`

### Experiment 3: Corner Injection

`exp3.py` compares ordinary uniform samples with matched samples where two observations are replaced by the all-negative and all-positive corners.

This is a mechanism test: if the plateau is caused by missing corner-aligned individuals, injecting those individuals should increase the observed LMO bound. The matched treatment uniforms keep the standard and corner-injected samples comparable.

Outputs:

- `outputs/experiment3_raw.csv`
- `outputs/experiment3_summary.csv`
- `outputs/experiment3_corner_summary.csv`
- `figures/gamma_corner_mc.png`

### Experiment 4: Covariate Geometry

`exp4.py` changes the marginal covariate distribution while keeping the same treatment-assignment mechanism.

It compares:

- `beta_u`: boundary-heavy covariates from `Beta(0.2, 0.2)` scaled to `[-1, 1]`.
- `uniform`: flat covariates on `[-1, 1]`.
- `beta_bell`: center-heavy covariates from `Beta(5.0, 5.0)` scaled to `[-1, 1]`.

The expected ordering is boundary-heavy > uniform > center-heavy, because boundary-heavy samples are more likely to contain individuals near the configurations that realize large omitted log-odds contributions.

Outputs:

- `outputs/experiment4_raw.csv`
- `outputs/experiment4_summary.csv`
- `outputs/experiment4_final_recovery.csv`
- `figures/gamma_3way_dgp_comparison.png`

## Running The Code

Install the Python dependencies:

```bash
pip install -r requirements.txt
```

Run the experiments from the repository root:

```bash
python exp1.py
python exp2.py
python exp3.py
python exp4.py
```

The full set of experiments runs in less than about 10-20 minutes total on the machine used for the thesis runs. Experiment 1 and the all-subset Monte Carlo experiments are the slowest because each `m` evaluates all feature combinations.

## Reproducibility

The checked-in `outputs/` and `figures/` are the generated results used in the thesis. Re-running the scripts overwrites those files with freshly generated summaries and plots.
